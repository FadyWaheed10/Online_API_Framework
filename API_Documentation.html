ğŸ‘¤ Customer APIs

Customer Endpoints

These APIs allow customers to manage their accounts, place orders, and interact with the system.

POST /customers
Create customer account or login via OTP (sends OTP to phone)
POST /customers/:customerId/verify
Verify OTP code sent to customer phone (returns customer token)
POST /resendsms
Resend OTP code to customer phone number
PATCH /customers/:customerId
Update customer profile information (name, email, etc.)
GET /menus
Browse available menus (can filter by concept or location)
GET /menus/:menuId/items
View menu items (can filter by category or loyalty rewards)
GET /locations
List available restaurant locations
POST /customers/:customerId/addresses
Add delivery address for customer
PATCH /customers/:customerId/addresses/:addressId
Update customer address
DELETE /customers/:customerId/addresses/:addressId
Delete customer address
POST /customers/:customerId/cars
Add vehicle information (for curbside pickup)
GET /customers/:customerId/cars
List customer vehicles
POST /orders
Create a new order (returns order ID and token for payment)
GET /orders/:id
Get order details and status
POST /promotions/validation
Validate order with promotions applied (returns discounted total)
GET /coupons
List available coupons
POST /orders/:orderId/payments
Create payment for an order
GET /payments/orders/ordertoken
Get payment details by order token
GET /orders/:id/employees
Get driver/employee assigned to order for tracking
GET /customers/:customerId/orders
Get order history for a customer
PATCH /orders/:id/cancellation
Cancel an existing order
GET /cms/pages
Get CMS pages (About, Terms, Privacy, etc.)
GET /cms/sliders
Get homepage banner sliders
GET /cms/sliders/:slider_id/slides
Get slides for a specific slider
POST /feedback
Submit customer feedback about order or service

Employee Endpoints

GET /orders/:id
View order details
GET /orders/:id/employees
Get driver details for an order

Authentication Requirements

No Authentication: Customer-facing endpoints (create customer, browse menu, create order)

Bearer Token: Employee endpoints (login required)

Bearer Token + Solo-Concept Header: Admin endpoints (login + concept key required)

API Key (X-API-Key Header): System endpoints (license management)

Common Headers

  • Authorization: Bearer {token} - For authenticated endpoints
  • Solo-Concept: {conceptKey} - For concept-specific operations
  • Solo-App: {appKey} - For guest login and some endpoints
  • X-API-Key: {apiKey} - For system/license endpoints
  • Content-Type: application/json - For POST/PATCH/PUT requests
  • Accept: application/json - For GET requests
